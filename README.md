# debaid

Debian packaging automation skills for Claude Code, shipped as a
plugin.

`debaid` is a Claude Code plugin that helps a Debian / Ubuntu
package maintainer bootstrap, modernise, lint-clean, and add
autopkgtest coverage to a package — while keeping the maintainer
in the loop for every judgement call.

The skills are **prescriptive** (house-style + policy + tooling
indicators) and **respectful** (default dry-run on destructive
operations, no uploads, no version bumps without explicit
request, no Maintainer-field changes).

## Language coverage

Workers specialise their output by detected upstream language.
Per-language overlays under `docs/references/languages/`:

- **Python** — pybuild, `dh-sequence-python3`, watch v5 with
  `Template: pypi`, autodep8 `Testsuite: autopkgtest-pkg-python`.
- **Rust** — dh-cargo for application binaries; debcargo
  bail-out for library crates; watch v5 with `Template: github`.
- **Go** — dh-golang, `XS-Go-Import-Path:`, vendored-dep
  stripping, watch v5 with `Template: github`.
- **Perl** — plain `dh $@`, autodep8
  `Testsuite: autopkgtest-pkg-perl`, watch v5 with
  `Template: metacpan`. *DRAFT pending pkg-perl maintainer
  review.*

For other languages, workers fall back to a generic `dh $@`
skeleton. See `docs/developer.md` § "Adding a language overlay"
for the recipe.

## Install

Claude Code plugins always install through a "marketplace" — but a
marketplace can be a local directory or a private git repo. Pick
the path that matches your situation.

### Per-session (dev / workshop)

Load the plugin from a checkout for the current Claude Code
session only. No marketplace needed:

```
claude --plugin-dir /path/to/debaid
```

Inside Claude Code, `/reload-plugins` picks up edits without
restart. This is what `workshop.yaml`'s `claude` action does.

### Persistent on one laptop

Create a one-plugin marketplace under `~/.local/share/` (or
wherever you like), then install from it:

```
mkdir -p ~/.local/share/claude-marketplaces/local/.claude-plugin

cat > ~/.local/share/claude-marketplaces/local/.claude-plugin/marketplace.json <<EOF
{
  "name": "local",
  "owner": { "name": "$(git config user.name)" },
  "plugins": [
    { "name": "debaid", "source": "/path/to/debaid" }
  ]
}
EOF
```

Then in Claude Code:

```
/plugin marketplace add ~/.local/share/claude-marketplaces/local
/plugin install debaid@local
```

`/debaid:run` and friends are now available from any project.
`/plugin update debaid@local` re-fetches after you pull new
commits.

If the install complains about an absolute `source`, fall back to
a symlink: put the marketplace at the same level as a symlink to
the plugin checkout, and set `source` to `"./debaid"` (relative).

### Team-shared (private git marketplace)

Push the marketplace JSON above to a private git repo. The plugin
`source` becomes the plugin's own git URL. Teammates run:

```
/plugin marketplace add <org>/<marketplace-repo>
/plugin install debaid@<marketplace-name>
```

For projects that should auto-load debaid on first open, add
this to the project's `.claude/settings.json`:

```json
{
  "extraKnownMarketplaces": {
    "<marketplace-name>": {
      "source": { "source": "github", "repo": "<org>/<marketplace-repo>" }
    }
  },
  "enabledPlugins": { "debaid@<marketplace-name>": true }
}
```

## Invocation

From within a source tree:

```
# Full pipeline
/debaid:run

# Single phase
/debaid:run --only=lintian
/debaid:lintian

# Phase subset
/debaid:run --only=refresh,lintian --skip=autopkgtest

# Dry-run (refresh defaults to this anyway)
/debaid:run --dry-run

# Custom house style
/debaid:run --house-style=/path/to/team-style.md

# Disable reference corpus
/debaid:run --reference=none

# Workshop mode: skip per-phase confirmation gates
/debaid:run --yes
```

## Design

See:

- `skills/run/SKILL.md` — orchestrator behaviour.
- `shared-context.md` — the contract every worker obeys
  (JSON context schema, verify-script output, iteration budget,
  bail-out format).
- `docs/house-style.md` — every prescriptive choice with a
  citation.

## What it will not do

- Upload anything (`dput`, `debrelease`, `dgit push`, `git push`).
- Edit `debian/changelog` distribution away from `UNRELEASED`.
- Edit `Maintainer:` or `Uploaders:`.
- Edit upstream sources in place (always a DEP-3 quilt patch).
- Suppress lintian tags without a `# reason:` comment.
- Set `Multi-Arch: same` without verifying file paths.

`.claude/settings.json` in this repo also ships a deny-list that
blocks the destructive operations above at the tool level, as
belt-and-braces with the prompt-level rules.
